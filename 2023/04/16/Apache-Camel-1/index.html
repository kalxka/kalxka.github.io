Camel基本了解

1.Camel Core提供的是创作项目的模板，使用各种maven原型
2.在配置好maven工具后，生成一个基于maven原型的项目my-app
3.pom.xml：项目配置核心文件
4.执行mvn package 打包；之后就可以编译打包的jar
命令一 java -cp target/my-app-1.0-SNAPSHOT.jar com.mycompany.app.App（指定主类）
命令二 java -jar test-mvn-package-1.0-SNAPSHOT.jar
mvn clean dependency:copy-dependencies package：清理项目，复制依赖项并打包项目
mvn site 基于pom上的信息生成了html

华为云

公有云也就是供应商提供计算、存储、数据库、各种服务资源，用户用这些资源可以更低成本地开发和维护。

OBS：对象存储服务。

基础介绍

选择OBS的理由：

  • 容量上：处理海量数据时，自建服务器存储容量限制于搭建的硬件设备，人工扩容非常麻烦，而OBS部署了很多数据中心，完全不必担心容量不够
  • 安全性上：自建服务器承担技术漏洞，错误操作和信息安全风险，OBS用SSL加密了用户数据，并用AS/SK服务对身份鉴权，多种方式保证传输和访问安全，对数据检查使数据持久性高

OBS应用场景：
网站和应用托管；大数据；云备份/归档
OBS使用方式：

  • OBS管理控制台：基于web可视化管理，通过网页管理数据
  • OBS客户端：操作软件，在个人电脑上数据管理
  • REST API：用提供的API接口和各种语言开发包SDK二次开发

详细介绍

产品架构：由桶（存储容器）和对象（存储单位）组成，理解为对象就是个文件夹，而桶里一堆这种文件夹

  • 桶：有存储类别、访问权限、所属区域等属性，用户通过桶的访问域名定位桶
  • 对象：一个文件的数据和相关属性信息的集合。包括key(对象名称),metadata(对象描述信息),data(数据)三个部分

存储类别：

  • 标准存储：适合频繁访问的，比如大数据，社交图片，移动应用
  • 低频访问存储：一年访问少于12次这种，适合企业备份
  • 归档存储：一年访问一次这种，适合长期备份
  • 深度归档存储：几年访问一次。
  • 上传对象时，对象继承桶的存储类别，但是也可以修改对象的存储类别
  • 修改桶的存储类别时，对象的不会改，之后新上传的对象会改变

OBS使用方式（REST API-API）

OBS API常用概念：

  • 账号:用户注册的账号，对资源和云服务有完全的访问权限，为了确保账号安全，一般建议创建用户使用他们进行管理
  • 用户:云服务使用人员，具有身份凭证，密码和访问密钥
  • 桶
  • 对象
  • 区域：云资源的物理位置，同一区域内可用内网互通，桶一旦创建，归属区域信息不能改变
  • 可用区：一个或多个物理数据中心的集合
  • 企业项目

SMN：消息通知服务

基本介绍

  • 应用场景：用户创建了一个主题，主题下有不同订阅者，不同订阅终端要发不同内容消息，使用JSON类信息，则可以实现向不同订阅者发不同消息
    （这里可以联想自己创建账号时收到邮件让你点击确认就是用的这种服务）
  • 功能：依据用户需求主动推送通知消息，用户通过短信、电子邮件方式接收（订阅者可以为邮箱地址、手机号码、函数及URL地址）
  • 优势：一键发送快速简便

SMN API

SMN API常用概念：

  • 组织：用户可以将多个华为云帐号整合到创建的组织中，集中管理
  • 帐号：用户注册时的帐号
  • 用户：由帐号在IAM中创建的用户，云服务使用人员，具有身份凭证（密码和访问密钥）
  • 区域：地理位置和网络时延维度划分，同一个区域内共享一些公共服务。区域分为通用区域（面向公共租户提供通用云服务）和专属区域（只承载同一类业务或只面向特定租户）
  • 可用区：一个或多个物理数据中心的集合
  • 项目：区域默认对应一个项目，这个项目由系统预设置，用来隔离物理区域内资源，想要精细的权限控制，可以在区域默认项目创建子项目，子项目中创建资源，以子项目为单位进行授权，使用户仅能访问特定子项目中资源，资源的权限控制更加精确
  • 企业项目：项目升级版，可以包含多个区域资源，项目中资源可以迁入迁出

DLI:数据湖探索

基本介绍

  • 优势：大数据处理分析服务，完全托管，使用门槛低，轻松使用SQL和SPARK程序，资源隔离不同部分安全访问，性能高
  • 步骤：
    实时更新来自全球不同平台的数据（包括资讯信息、交易数据、用户数据和行情信息）
    通过数据接入服务转储数据（数据接入）
    实时展示数据，多维度的展示（数据存储处理和分析）
    精准分析等多元化服务（业务应用）

详细介绍

数据湖探索：
完全兼容Apache Spark、Apache Flink、openLooKeng生态，提供一站式的流处理、批处理、交互式分析的Serverless融合处理分析服务。
支持SQL/Spark SQL/Flink SQL,使用SQL或程序就可以对各种数据库探索
功能介绍：

  • 三大基本功能：
    1.SQL作业：提供标准SQL语句
    2.Flink作业：支持多种函数，用SQL表达业务逻辑
    3.Spark作业：通过交互式会话session和批处理batch提交计算任务
  • 多数据源分析：
    Spark跨源连接
    Flink跨源支持与多种云服务连通
  • 存算分离：
    数据存储到OBS，DLI可以直接和OBS对接进行数据分析
    分为单AZ存储和多AZ存储
    单AZ存储：数据存储在单个AZ
    多AZ存储：数据存储在多个AZ，选择多AZ存储的桶
  • 弹性资源池：
    有的时间段对资源需求大，因此分时按需弹性，不同时间段对资源扩容缩容
  • DLI核心引擎：
    Spark（大规模数据统一分析引擎）+Flink（分布式计算引擎）+openLooKeng（SQL引擎Presto）
  • DLI服务框架：
    Serverless：无服务器化
  • 关于openLooKeng(自研交互式分析presto):
    数据接入后Flink实时分析再通过OpenLooKeng得到在线应用，数据大屏和运营看板

使用

  • web化的服务管理平台，管理控制台管理
  • API方式二次开发
  • JDBC或ODBC连接服务端
  • Beeline
  • Spark-submit
  • 数据治理中心DataArts Studio

GaussDB NoSQL:云数据库

基本介绍

NoSQL非关系型数据库，存算分离的分布式多模NoSQL数据库服务，
for redis:特点是数据都存在磁盘而不是内存中，容量更大，成本更低

典型应用

  • 游戏应用可以将游戏数据存储其中，比如用户装备，游戏玩家活跃高峰期，快速灵活添加计算节点，快速扩容
  • 电子商务网站和娱乐网站，存储访问者的活动，可以结合Spark大数据分析，用于实时推荐等
  • 金融行业结合Spark等大数据分析工具，用于金融行业的风控体系

DMS：分布式消息服务

基本介绍

高可用分布式集群技术的消息中间件服务，提供接口，提供中间件服务，和SMN服务不同

DEW：数据加密服务

基本介绍

提供四个微服务：密钥管理服务KMS，云凭据管理服务CSMS，密钥对管理服务KPS，专属加密DHSM
